Walkthrough - 2FA & Security Upgrade
I have implemented Two-Factor Authentication (TOTP) and secure password hashing (bcrypt) for your Timesheet application.

🔐 Security Enhancements
1. Two-Factor Authentication (TOTP)
Users now have an extra layer of security compatible with Google Authenticator, Authy, or any TOTP-compliant app.

Setup Phase: On the first login after this update, users will be presented with a QR Code. Scanning this adds the account to their mobile authenticator app.
Verification Phase: Subsequent logins will require a 6-digit Time-based One-Time Password (OTP) after entering the correct username and password.
2. Secure Password Hashing
Bcrypt Hashing: Passwords are no longer stored in plain text. They are hashed using the industry-standard bcrypt algorithm.
Auto-Upgrade: For a seamless transition, existing plain-text passwords will be automatically upgraded to secure hashes the first time a user logs in successfully.

🚀 How to Test
Restart the App: Ensure dependencies are installed (pip install -r requirements.txt).
Login: Use your existing credentials (e.g., "System Administrator" / "12345678").
Setup 2FA:
Scan the QR code shown on the screen.
Enter the 6-digit code from your phone.
Click "Verify & Enable 2FA".
Confirm: Log out and log back in. You should now be asked for an OTP immediately after entering your password.
